Part I — CLI Reference Guide

A comprehensive command-line interface for the Advanced RVC Inference framework. This CLI provides access to all the powerful features of RVC including voice conversion, model training, audio separation, and more — all from the terminal.

Installation

From Source

# Clone the repository
git clone https://github.com/ArkanDash/Advanced-RVC-Inference.git
cd Advanced-RVC-Inference

# Install the package
pip install -e .

# Or install with all dependencies
pip install -e .[all]

Using the CLI Directly

# Run directly with Python
python rvc_cli.py --help

# Or use the wrapper script (Linux/macOS)
./rvc-cli --help

# Make the wrapper executable (Linux/macOS)
chmod +x rvc-cli

Quick Start

Basic Voice Conversion

# Convert voice with default settings
rvc-cli infer -m mymodel.pth -i input.wav -o output.wav

# Convert with pitch shift (12 semitones = octave up)
rvc-cli infer -m mymodel.pth -i input.wav -p 12

# Convert with custom F0 method
rvc-cli infer -m mymodel.pth -i input.wav --f0_method harvest

Audio Separation

# Separate vocals from music
rvc-cli uvr -i music.wav

# Separate with specific model
rvc-cli uvr -i music.wav --model MDXNET_Main --output ./separated

Launch Web Interface

# Start web interface on default port
rvc-cli serve

# Start on custom port with public share
rvc-cli serve --port 8080 --share

infer — Voice Conversion

Convert voice in an audio file using an RVC model.

rvc-cli infer -m <model> -i <input> [options]

Required Arguments

-m, --modelPath to RVC model file (.pth or .onnx)
-i, --inputPath to input audio file

Optional Arguments

-o, --outputOutput file path (auto-generated if not specified)
-p, --pitchPitch shift in semitones (default: 0)
-f, --formatOutput format — wav, mp3, flac, ogg (default: wav)
--indexPath to .index file for better quality
--f0_methodF0 extraction method (default: rmvpe)
--filter_radiusFilter radius for F0 smoothing (default: 3)
--index_rateIndex strength 0.0-1.0 (default: 0.5)
--rms_mix_rateRMS mix rate (default: 1.0)
--protectProtect consonants 0.0-1.0 (default: 0.33)
--hop_lengthHop length for processing (default: 64)
--embedder_modelEmbedding model (default: hubert_base)
--resample_srResample rate (0 = original, default: 0)
--split_audioSplit audio before processing
--checkpointingEnable memory checkpointing
--f0_autotuneEnable F0 autotune
--f0_autotune_strengthAutotune strength (default: 1.0)
--formant_shiftingEnable formant shifting
--formant_qfrencyFormant frequency coefficient (default: 0.8)
--formant_timbreFormant timbre coefficient (default: 0.8)
--clean_audioApply audio cleaning
--clean_strengthCleaning strength (default: 0.7)
Example
rvc-cli infer -m artist_model.pth -i speech.wav -o converted.wav -p 2 \
    --index_rate 0.75 --f0_method rmvpe --clean_audio

Examples & Workflows

Complete Training Workflow

# 1. Create dataset from YouTube
rvc-cli create-dataset -u "https://youtube.com/watch?v=xxx" \
    --output ./dataset --sample_rate 48000 --separate

# 2. Preprocess data
rvc-cli preprocess mymodel --sample_rate 48000 --cut_method Automatic

# 3. Extract features
rvc-cli extract mymodel --sample_rate 48000 --f0_method rmvpe --gpu 0

# 4. Train model
rvc-cli train mymodel --version v2 --epochs 300 --batch_size 8 --gpu 0

# 5. Create index (after training)
rvc-cli create-index mymodel --version v2

Voice Conversion with Different F0 Methods

# Using RMVPE (recommended)
rvc-cli infer -m model.pth -i input.wav -o output_rmvpe.wav --f0_method rmvpe

# Using Harvest (faster)
rvc-cli infer -m model.pth -i input.wav -o output_harvest.wav --f0_method harvest

# Using Crepe (most accurate but slow)
rvc-cli infer -m model.pth -i input.wav -o output_crepe.wav --f0_method crepe-medium

Batch Processing

# Process all audio files in a directory
for file in ./inputs/*.wav; do
    rvc-cli infer -m model.pth -i "$file" -o "./outputs/$(basename $file)"
done

Troubleshooting

"Model file not found"

Ensure the model path is correct. Check that the model file has .pth or .onnx extension. Verify file permissions.

"CUDA out of memory"

Reduce batch size: --batch_size 4. Enable checkpointing: --checkpointing. Use CPU: --gpu -.

"Audio format not supported"

Convert audio to WAV first: ffmpeg -i input.mp3 output.wav. Supported formats: wav, mp3, flac, ogg, opus, m4a, aac.

"F0 method not available"

Install ONNX runtime for some methods. Some methods require specific embedders.

Tier 1: Best Quality

HiFi-GAN (Default)

⭐⭐⭐⭐⭐
Category: HiFi-GANSource: models/generators/hifigan.pyRating: 5.0/5

HiFi-GAN is the standard and default vocoder. It uses transposed convolution upsampling with weight-normalized residual blocks (Multi-Receptive Field fusion). Each upsampling layer is followed by multiple residual blocks with different kernel sizes and dilation rates, capturing features at multiple temporal scales. Lightweight and fast while still producing high-quality audio output. Works with both pitch-guided and non-pitch-guided training modes.

  • Standard transposed convolution upsampling
  • Weight-normalized residual blocks with MRF fusion
  • Works with and without pitch guidance (f0)
  • Lightweight and fast inference

Recommended for: Default choice for all training. Best balance of quality, speed, and compatibility.

BigVGAN

⭐⭐⭐⭐⭐
Category: Anti-Aliased GANPaper: 2023Rating: 5.0/5

The highest-quality vocoder available. Introduces Snake activations with Anti-Aliasing (SnakeBeta and AMP blocks) and data-augmented adversarial training. Snake provides a periodic nonlinearity naturally suited for audio signals, while anti-aliased design prevents high-frequency artifacts. Uses kaiser-sinc filters for both upsampling and downsampling, achieving state-of-the-art quality across benchmarks.

  • SnakeBeta learnable periodic activations
  • Anti-Aliased Multi-Period (AMP) blocks
  • Kaiser-sinc filters for upsampling/downsampling
  • Superior high-frequency reconstruction

Recommended for: Maximum audio quality. Best for singing voice conversion and high-fidelity speech synthesis.

Default (HiFi-GAN NSF)

⭐⭐⭐⭐⭐
Category: HiFi-GANSource: models/generators/nsf_hifigan.pyRating: 5.0/5

HiFi-GAN with Neural Sine Filter (NSF). Combines HiFi-GAN's upsampling with a Neural Sine Filter that injects harmonic information directly into each upsampling layer. The NSF source module generates sine waves conditioned on F0, mixed through noise convolution layers. Provides improved pitch accuracy due to explicit harmonic conditioning, but requires pitch guidance (f0) to be enabled.

  • Neural Sine Filter for harmonic injection
  • Noise convolutions at each upsampling layer
  • Improved pitch accuracy from explicit harmonic conditioning
  • Supports gradient checkpointing; requires pitch guidance (f0)

Recommended for: Users who need improved pitch accuracy. The explicit harmonic injection helps with singing voice and tonal languages.

Tier 2: Excellent Quality

HiFi-GAN-v3

⭐⭐⭐⭐½
Category: Enhanced HiFi-GANRating: 4.5/5

Enhanced variant incorporating SnakeBeta activations and wider dilation patterns. Unlike original HiFi-GAN which uses LeakyReLU, v3 uses the same Snake activation that made BigVGAN successful, providing periodic nonlinearity better matching audio signal characteristics. Wider dilation captures longer-range temporal dependencies for improved formant transition modeling.

  • SnakeBeta learnable activations
  • Enhanced residual blocks with wider dilations
  • Gradient checkpointing support
  • Drop-in upgrade over standard HiFi-GAN

Recommended for: Quality upgrade over Default without the computational cost of BigVGAN.

MRF-HiFi-GAN

⭐⭐⭐⭐½
Category: Multi-Receptive FieldRating: 4.5/5

Replaces standard residual blocks with Multi-Receptive Field (MRF) blocks. Each MRF block contains a sequence of MRFLayers with different dilation stacks, capturing features at multiple temporal scales simultaneously. Particularly effective for speech with complex spectral characteristics. SineGenerator provides harmonic conditioning with 8 harmonics.

  • Multi-Receptive Field fusion blocks
  • Multiple dilation stacks per block
  • SineGenerator with 8 harmonics
  • Gradient checkpointing support

Recommended for: Speech with complex spectral characteristics. Good for multi-speaker models.

Vocos

⭐⭐⭐⭐½
Category: Fourier-basedRating: 4.5/5

Fundamentally different approach using inverse Short-Time Fourier Transform (iSTFT) for waveform reconstruction instead of transposed convolution upsampling. Inherently more lightweight because it operates in the frequency domain. The backbone uses 1D convolutions with Snake activations, and the iSTFT head predicts both magnitude and phase for clean waveform output.

  • iSTFT-based waveform reconstruction (no transposed convolutions)
  • Lightweight and computationally efficient
  • Magnitude + phase prediction
  • Naturally suited for harmonic signals

Recommended for: Fast inference and lightweight models. Excellent for real-time applications.

RingFormer

⭐⭐⭐⭐½
Category: Conformer-basedRating: 4.5/5

The most architecturally advanced vocoder, using Conformer blocks with both time and frequency attention mechanisms. Uses einops-based rearrange operations for efficient multi-head attention. Output layer uses iSTFT for waveform generation from predicted magnitude and phase. Uses ResBlock_SnakeBeta and SourceModuleHnNSF with 8 harmonics for harmonic conditioning.

  • Conformer attention (time + frequency)
  • iSTFT output with magnitude + phase prediction
  • Most complex architecture; unique 3-output return
  • Higher memory usage than other vocoders

Recommended for: Advanced users seeking maximum quality through attention-based processing.

Tier 3: Very Good Quality

JVSF-HiFi-GAN

⭐⭐⭐⭐
Category: Source-FilterRating: 4.0/5

Joint-Variable Source-Filter explicitly separates voice generation into source (excitation) and filter (vocal tract) components, mirroring the human speech production model. The source module generates learnable harmonic components, while the filter module uses a pyramid of dilated convolutions. This separation allows independent control of pitch-related and timbre-related features.

Recommended for: Voice conversion where preserving speaker identity is critical.

RefineGAN

⭐⭐⭐⭐
Category: U-Net GANRating: 4.0/5

U-Net architecture with skip connections. The harmonic downsampling path processes F0 through sine generation and progressive downsampling. The upsampling path uses ParallelResBlocks with three parallel branches (kernel sizes 3, 7, 11) combined through AdaIN noise injection. Skip connections preserve fine spectral details.

Recommended for: High-fidelity audio where spectral detail preservation is important.

NSF-APNet

⭐⭐⭐⭐
Category: Hybrid NSFRating: 4.0/5

Combines Neural Sine Filter with an All-Pass Network for improved phase modeling. The All-Pass Network applies magnitude-preserving phase corrections at each residual block, addressing inaccurate phase reconstruction common in GAN-based vocoders. Particularly effective in higher frequency ranges where phase errors are most perceptible.

Recommended for: Audio with prominent high-frequency content. Reduces metallic artifacts.

FullBand-MRF

⭐⭐⭐⭐
Category: Enhanced MRFRating: 4.0/5

Extends MRF-HiFi-GAN with significantly wider dilation patterns (up to rate 15) and channel attention mechanisms. Wider dilations capture longer-range dependencies for better formant transition modeling. Channel attention helps focus on the most informative frequency bands dynamically. Snake activations match the quasi-periodic nature of speech.

Recommended for: Speech with rapid formant transitions or wide pitch ranges.

Tier 4: Good Quality

WaveGlow

⭐⭐⭐½
Category: Flow-basedRating: 3.5/5

Flow-based vocoder using invertible 1x1 convolutions and WaveNet-style layers with gated activations. Unlike GAN-based vocoders, WaveGlow learns the exact probability distribution of audio waveforms through maximum likelihood estimation with invertible transformations. Provides more stable training but requires more computation.

Recommended for: Users who prefer deterministic models. Avoids adversarial training instability.

PCPH-GAN

⭐⭐⭐½
Category: Phase-CorrectedRating: 3.5/5

Phase-Corrected Parallel HiFi-GAN uses PCHIP interpolation for F0 upsampling, providing smoother pitch contour transitions. The PCPH generator models pitch harmonics with a specialized source module, while SiLU and SnakeBeta provide smooth nonlinearity. A progressive upsampling path creates a coarse-to-fine synthesis pipeline.

Recommended for: Singing voice conversion where smooth pitch transitions are important.

Recommendations for RVC Training

Beginner

Use HiFi-GAN (the default). Lightweight, fast, and works with both pitch-guided and non-pitch-guided training. The best starting point for all users.

Intermediate

Try BigVGAN for the highest quality, Default (HiFi-GAN NSF) for improved pitch accuracy, or HiFi-GAN-v3 for a lighter upgrade.

Advanced

Experiment with RingFormer for attention-based synthesis, Vocos for lightweight inference, or MRF-HiFi-GAN for multi-scale feature extraction.

Real-time / Low Latency

Use Vocos — its Fourier-based architecture is inherently faster at inference since it avoids expensive transposed convolution upsampling.

Maximum Quality

Use BigVGAN — it consistently achieves the highest objective and subjective quality scores across all benchmarks.

Technical Notes
  • Non-Default/HiFi-GAN vocoders require v2 + pitch guidance (enforced by UI)
  • Non-Default/HiFi-GAN vocoders use fp32 at inference (fp16 disabled for stability)
  • Vocoder choice is saved in the model checkpoint and used automatically during inference
  • Pre-trained weights for non-HiFi-GAN vocoders follow the naming pattern: {VocoderName}_f0G48k.pth
  • All vocoders support gradient checkpointing except BigVGAN and WaveGlow
  • The vocoder registry is in arvc/models/generators/__init__.py

Tier 1: Best for RVC/Audio Training

AdamW

⭐⭐⭐⭐⭐
Category: PyTorch Built-inPaper: 2019Rating: 5.0/5

Adam with decoupled weight decay is the gold standard optimizer for deep learning training. It combines adaptive learning rate with proper L2 regularization by decoupling weight decay from the gradient update. The default and recommended optimizer for RVC model training, providing reliable convergence across all architectures, dataset sizes, and training configurations.

  • Adaptive learning rates per parameter
  • Decoupled weight decay (proper L2 regularization)
  • Fused CUDA kernel support for faster training
  • Proven track record across all of deep learning

Recommended for: All RVC training scenarios as the default. Works well with LR 1e-4 to 1e-3, batch sizes 4-32, and 100-1000 epochs.

ScheduleFreeAdamW

⭐⭐⭐⭐⭐
Category: Schedule-FreePaper: 2024Rating: 5.0/5

Eliminates the need for any learning rate scheduling by maintaining dual "z" (lookahead) and "y" (standard AdamW) parameters. The optimizer dynamically adjusts its effective learning rate based on the distance between z and y, providing built-in warmup at the start and natural decay as convergence approaches. Never worry about warmup steps, cosine annealing, or step decay again.

  • No learning rate schedule needed whatsoever
  • Built-in warmup phase (first ~5% of training)
  • Automatic decay as training converges
  • Drop-in replacement for AdamW

Recommended for: Users who want to avoid LR schedule tuning. Especially useful with varying dataset sizes.

Muon

⭐⭐⭐⭐⭐
Category: Second-OrderPaper: 2025Rating: 5.0/5

Applies Newton-Schulz iteration to orthogonalize the momentum vector at each step. This normalization provides significantly better conditioning for the optimization landscape, similar to preconditioning in second-order methods but at much lower cost. Demonstrates faster convergence compared to AdamW, particularly in later training stages.

  • Momentum orthogonalization via Newton-Schulz iteration
  • Better conditioned optimization landscape
  • Faster convergence on deep models
  • Works well with high learning rates

Recommended for: Advanced users training large RVC models (v2, 48k) wanting faster convergence. Effective with 300+ epoch training runs.

Sophia

⭐⭐⭐⭐⭐
Category: Second-OrderPaper: 2023Rating: 5.0/5

Second-order optimizer using a diagonal Hessian estimate combined with stochastic clipping. Unlike Adam which only uses first-order gradient information, Sophia incorporates curvature information from the Hessian (second derivatives) for more informed update decisions. The diagonal approximation keeps memory manageable while the clipping mechanism prevents excessively large updates in high-curvature directions.

  • Diagonal Hessian estimation for curvature awareness
  • Stochastic clipping for stability
  • Faster convergence than first-order methods
  • Memory-efficient diagonal approximation

Recommended for: Users with sufficient GPU memory wanting maximum convergence speed. Best with larger batch sizes (8+) and longer training runs.

Tier 2: Excellent Optimizers

Lion

⭐⭐⭐⭐½
Category: Sign-BasedPaper: 2023Rating: 4.5/5

Discovered through automated program search. Uses the sign of the momentum rather than the momentum itself for the update direction. This dramatically simplifies computation and results in ~50% less memory than AdamW (only one state tensor vs. two). Often matches or exceeds AdamW performance, particularly with higher learning rates.

Recommended for: Memory-constrained training or when using higher learning rates than AdamW allows.

Prodigy

⭐⭐⭐⭐½
Category: LR-FreePaper: 2023Rating: 4.5/5

Automatically determines the optimal learning rate by estimating the distance to the solution (D0) using gradient statistics. You only need to set d_coef. The optimizer continuously adapts its effective LR during training, eliminating the most common failure mode — choosing the wrong learning rate.

Recommended for: Users who struggle with LR tuning or need a "set it and forget it" optimizer across different model architectures.

NAdam

⭐⭐⭐⭐½
Category: PyTorch Built-inPaper: 2015Rating: 4.5/5

Combines Adam's adaptive learning rates with Nesterov accelerated gradient. The Nesterov aspect means the optimizer looks ahead by computing the gradient at the anticipated next position, providing implicit momentum correction that often leads to faster convergence, especially in early training stages.

Recommended for: A slight upgrade over AdamW without the complexity of newer optimizers. Good default alternative.

Tier 3: Very Good Optimizers

RAdam

⭐⭐⭐⭐
Category: PyTorch Built-inRating: 4.0/5

Rectified Adam dynamically switches between SGD-like updates (when variance is unreliable) and Adam-like updates (when variance becomes trustworthy), eliminating the need for warmup steps.

Recommended for: Short training runs where warmup would consume a significant fraction of total steps.

Adan

⭐⭐⭐⭐
Category: NesterovRating: 4.0/5

Introduces a unique third moment that tracks the difference between consecutive gradients, capturing curvature information at first-order cost. Has shown particularly strong results on vision and audio tasks.

Recommended for: Audio/vision training tasks where gradient smoothness matters.

Ranger21

⭐⭐⭐⭐
Category: CombinedRating: 4.0/5

Synergistically combines RAdam's variance rectification with Lookahead's slow-fast weight synchronization. Every k steps, interpolates between "fast" weights (RAdam) and "slow" weights. Acts as a regularizer that prevents overshooting minima.

Recommended for: A "best of both worlds" optimizer with RAdam's stability and Lookahead's generalization benefits.

AdaFactor

⭐⭐⭐⭐
Category: Memory-EfficientRating: 4.0/5

Dramatically reduces memory usage by factoring the second-moment estimator into row-wise and column-wise statistics. For a (m, n) parameter matrix, Adam stores m x n variance values while AdaFactor stores m + n. Instrumental in training T5.

Recommended for: Training large RVC models on GPUs with limited memory.

DAdaptAdam

⭐⭐⭐⭐
Category: LR-FreeRating: 4.0/5

Automatically determines the learning rate by estimating the distance to the optimal solution from accumulated gradient statistics. Maintains the Adam update rule but automatically adjusts the effective LR. Set lr=1.0 and let D-Adapt handle the rest.

Recommended for: Automatic LR tuning while keeping familiar Adam behavior.

PAdam

⭐⭐⭐⭐
Category: Partial AdaptiveRating: 4.0/5

Introduces a p_partial parameter that controls how much of the second moment's power to use. At p_partial=0 it behaves like SGD; at p_partial=1 it's Adam. Default 0.25 provides a balance retaining some adaptivity while gaining generalization benefits.

Recommended for: A balance between Adam's fast convergence and SGD's good generalization.

Apollo

⭐⭐⭐⭐
Category: Quasi-NewtonRating: 4.0/5

Approximates diagonal Hessian information using the ratio of consecutive gradients, similar to L-BFGS. Provides second-order convergence benefits without full Hessian computation. Starts with Adam-like behavior and progressively incorporates more curvature information.

Recommended for: Quasi-Newton convergence speed without the complexity and memory cost of full second-order methods.

Recommendations for RVC Training

Beginner

Start with AdamW (default). Use learning rate 1e-3 with 300 epochs and batch size 8.

Intermediate

Try ScheduleFreeAdamW to eliminate LR schedule tuning, or NAdam for slightly faster convergence. Drop-in replacements requiring no additional configuration.

Advanced

Experiment with Sophia or Muon for faster convergence on larger models. Prodigy and DAdaptAdam eliminate LR tuning entirely.

Memory-Constrained

Use Lion (50% less memory than Adam) or AdaFactor (sublinear memory scaling).

Large-Batch Training

Use LAMB or LARS for per-layer adaptive learning rate scaling, preventing gradient explosion in large-batch scenarios.

Technical Notes
  • All custom optimizers are implemented in arvc/models/optimizers/
  • The central registry in __init__.py maps optimizer names to their classes
  • The training engine uses the registry for dynamic optimizer selection
  • Each optimizer automatically receives appropriate kwargs (betas, eps, weight_decay) based on its capabilities
  • Fused CUDA kernels are automatically enabled when supported (currently only AdamW)
  • For optimizers that don't support betas or eps, these parameters are silently omitted
